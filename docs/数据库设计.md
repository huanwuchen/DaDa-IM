# DaDa IM 数据库设计

## 一、数据库概览

**数据库引擎**: Room (SQLite)
**数据库版本**: 5
**数据库类**: `AppDatabase`

```kotlin
@Database(
    entities = [
        ChatMessageEntity::class,      // AI 聊天消息
        ImContactEntity::class,        // 通讯录
        ImConversationEntity::class,   // 会话列表
        ImMessageEntity::class,        // IM 消息
        ImUserProfileEntity::class,    // 用户资料
    ],
    version = 5,
    exportSchema = false,
)
abstract class AppDatabase : RoomDatabase()
```

证据:
- 文件: `core/database/src/main/java/com/dada/core/database/AppDatabase.kt`

## 三、核心表详解

### 3.1 im_messages — IM 消息表

**职责**: 存储所有 IM 单聊消息

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| seq | Long | PK, AUTO_INCREMENT | 本地排序序号，保证同一毫秒多条消息有序 |
| id | String | UNIQUE, NOT NULL | 业务消息 ID (UUID)，用于去重 |
| conversationId | Long | NOT NULL, INDEX | 会话归属 = 对方 userId |
| fromId | Long | NOT NULL | 发送者 ID |
| toId | Long | NOT NULL | 接收者 ID |
| content | String | NOT NULL | 消息内容 / 媒体 URL |
| type | String | NOT NULL | 消息类型: text/image/video/audio/file/call_hint |
| timestamp | Long | NOT NULL | 消息时间戳 (ms) |
| isMine | Boolean | NOT NULL | 是否自己发的 |
| thumbUrl | String? | — | 缩略图 URL |
| duration | Long | DEFAULT 0 | 音视频时长 (ms) |
| size | Long | DEFAULT 0 | 文件大小 (bytes) |
| width | Int | DEFAULT 0 | 图片/视频宽度 |
| height | Int | DEFAULT 0 | 图片/视频高度 |
| fileName | String? | — | 原始文件名 |
| avatar | String? | — | 发送者头像 URL |
| iconRes | Int | DEFAULT 0 | 通话提示图标资源 ID |

**索引**:
```sql
CREATE UNIQUE INDEX index_im_messages_id ON im_messages (id)
CREATE INDEX index_im_messages_conversationId_timestamp_seq
    ON im_messages (conversationId, timestamp, seq)
```

**设计要点**:
1. `seq` 自增主键作为二级排序键，解决同一毫秒多条消息的排序问题
2. `conversationId = 对方 userId`，简化会话归属计算
3. 媒体字段共用 `content` 存 URL，扩展字段按需使用
4. `IGNORE` 冲突策略防止 WebSocket 回显导致重复

证据:
- 文件: `core/database/src/main/java/com/dada/core/database/entity/ImMessageEntity.kt`
- 文件: `core/database/src/main/java/com/dada/core/database/dao/ImMessageDao.kt`

### 3.2 im_conversations — 会话表

**职责**: 存储会话列表，支持首页展示

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| peerId | Long | PK | 对方用户 ID |
| peerUsername | String | NOT NULL | 对方昵称 (冗余存储) |
| peerAvatar | String? | — | 对方头像 (冗余存储) |
| lastMessage | String | DEFAULT "" | 最后一条消息内容 |
| lastMessageTime | Long | NOW | 最后消息时间戳 (排序用) |
| lastMessageType | String | "text" | 最后消息类型 |
| unreadCount | Int | DEFAULT 0 | 未读消息数 |

**设计要点**:
1. `peerId` 直接做主键，一个对方只有一条会话记录
2. 冗余存储 `peerUsername` 和 `peerAvatar`，避免查询会话列表时 JOIN 联系人表
3. `lastMessageTime` 用于倒序排序
4. `unreadCount` 支持未读角标

证据:
- 文件: `core/database/src/main/java/com/dada/core/database/entity/ImConversationEntity.kt`
- 文件: `core/database/src/main/java/com/dada/core/database/dao/ImConversationDao.kt`

### 3.3 im_contacts — 通讯录表

**职责**: 缓存联系人/在线用户信息

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Long | PK | 用户 ID |
| deviceId | String | DEFAULT "" | 设备 ID |
| username | String | NOT NULL | 昵称 |
| avatar | String? | — | 头像 URL |
| coverImage | String? | — | 朋友圈封面图 |
| online | Boolean | DEFAULT false | 是否在线 |
| updateTime | Long | NOW | 本地更新时间 (过期判断) |

**设计要点**:
1. `replaceAll()` 方法用事务保证原子性：先清空再批量写入
2. `updateTime` 用于判断缓存是否过期

## 四、索引设计

| 表 | 索引 | 类型 | 用途 |
|----|------|------|------|
| im_messages | (id) | UNIQUE | 消息去重 |
| im_messages | (conversationId, timestamp, seq) | COMPOSITE | 会话消息排序查询 |

**查询场景分析**:
- 查看某个会话的消息: `WHERE conversationId = ? ORDER BY timestamp ASC, seq ASC` → 命中复合索引
- 消息去重: `WHERE id = ?` → 命中唯一索引
- 会话列表: `ORDER BY lastMessageTime DESC` → 主键扫描 (数据量不大)

## 七、性能优化

1. **复合索引**: `(conversationId, timestamp, seq)` 覆盖会话消息查询
2. **IGNORE 冲突策略**: 防止重复插入，不抛异常
3. **Flow 响应式**: Room Flow 自动通知 UI，无需手动刷新
4. **事务批量操作**: `ImContactDao.replaceAll()` 用 `@Transaction` 保证原子性
5. **MMKV**: 轻量 KV 操作使用 MMKV，比 SharedPreferences 快 100 倍
6. **冗余存储**: 会话表冗余存储对方昵称/头像，避免 JOIN 查询
